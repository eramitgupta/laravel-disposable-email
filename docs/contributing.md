# Contributing

If you want to improve the package, fix a bug, expand the docs, or help maintain the disposable domain sources, contributions are welcome.

## Good ways to contribute

- Report incorrect validation behaviour
- Suggest improvements to the API or docs
- Add better examples for real Laravel apps
- Improve domain source handling and sync behaviour
- Help with performance and testing

## Before opening a pull request

Try to keep the change focused and easy to review. If you are changing behaviour, explain the problem clearly and describe the expected result.
